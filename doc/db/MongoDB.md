# 文档数据库
非关系型数据库（NoSQL）是指不采用传统关系模型（表格+行+列+SQL）的数据库，设计上更强调水平扩展性、灵活的数据模型和高性能读写。
以MongoDB为例
## 核心概念
* Database: 库
* Collection: 集合，类似表
* Document: 文档，基本存储单元，BSON格式
* Field: 字段
* Index: 索引，单字段索引、复合索引、多键索引
## 常用场景
1. 内容管理
2. 大量json数据
3. 电商产品目录
4. 用户资料
## 不适用场景
1. 复杂的多表关联事务
2. 关系型数据
3. 复杂的数据分析和计算
4. 搜索和模糊查询
5. 复杂图关系查询
6. 大量小文件存储
